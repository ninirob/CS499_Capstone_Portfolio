3-2 Assignment: Identifying CIFAR-10 Images — Enhanced (CS 499 Milestone Three)¶

This notebook enhances the original CS 370 CIFAR-10 CNN assignment for the CS 499 capstone, Category Two: Algorithms and Data Structure.

Fixes applied:

  • Corrected a Conv2D(64, 3, 3) positional-argument bug in the deeper network that was silently applying an unintended stride of 3 instead of a (3,3) kernel with default stride.
  • Corrected a stale-evaluation bug where the augmented model's test score was never captured, so the printed results were actually the earlier baseline model's leftover values.

Enhancements added:

  • Batch normalization after each convolutional layer.
  • ReduceLROnPlateau learning-rate scheduling during training.
  • A before/after convergence comparison isolating the effect of these two enhancements on the same architecture and data.

Deeper Network Architecture¶

Built as a function so the same architecture can be instantiated with or without batch normalization, which lets the before/after comparison later isolate the effect of that single enhancement.

Bug fix: the second 64-filter convolution originally read Conv2D(64, 3, 3), which Keras parses as kernel_size=3, strides=3 rather than the intended (3,3) kernel with default stride — silently breaking the downsampling and dropping the padding='same' used by every sibling layer. It is corrected below to Conv2D(64, (3, 3), padding='same').

In [10]:
def build_deeper_model(batch_norm=False):
    model = Sequential()

    model.add(Conv2D(32, (3, 3), padding='same', input_shape=(IMG_ROWS, IMG_COLS, IMG_CHANNELS)))
    if batch_norm:
        model.add(BatchNormalization())
    model.add(Activation('relu'))

    model.add(Conv2D(32, (3, 3), padding='same'))
    if batch_norm:
        model.add(BatchNormalization())
    model.add(Activation('relu'))
    model.add(MaxPooling2D(pool_size=(2, 2)))
    model.add(Dropout(0.25))

    model.add(Conv2D(64, (3, 3), padding='same'))
    if batch_norm:
        model.add(BatchNormalization())
    model.add(Activation('relu'))

    model.add(Conv2D(64, (3, 3), padding='same'))  # FIXED: was Conv2D(64, 3, 3)
    if batch_norm:
        model.add(BatchNormalization())
    model.add(Activation('relu'))
    model.add(MaxPooling2D(pool_size=(2, 2)))
    model.add(Dropout(0.25))

    model.add(Flatten())
    model.add(Dense(512))
    model.add(Activation('relu'))
    model.add(Dropout(0.5))
    model.add(Dense(NB_CLASSES))
    model.add(Activation('softmax'))
    return model

build_deeper_model(batch_norm=True).summary()

def make_augmented_dataset(datagen, X, Y, batch_size):
    ds = tf.data.Dataset.from_generator(
        lambda: datagen.flow(X, Y, batch_size=batch_size, shuffle=True),
        output_signature=(
            tf.TensorSpec(shape=(None, IMG_ROWS, IMG_COLS, IMG_CHANNELS), dtype=tf.float32),
            tf.TensorSpec(shape=(None, NB_CLASSES), dtype=tf.float32),
        )
    )
    return ds.repeat()

After: Deeper Network, With Batch Normalization and ReduceLROnPlateau¶

Same architecture and same augmented data, this time with batch normalization enabled and a ReduceLROnPlateau callback that halves the learning rate whenever validation loss stalls for 3 consecutive epochs. This is the "after" side of the comparison.

Bug fix: the original code commented out score = model.evaluate(...) after this training block, so the print statements below it were reporting the earlier baseline model's leftover score variable instead of this model's actual results. score_after is captured explicitly below to fix that.

model_after = build_deeper_model(batch_norm=True)

history_after = model_after.fit(
